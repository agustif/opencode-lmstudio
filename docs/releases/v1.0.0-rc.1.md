# opencode-lmstudio v1.0.0-rc.1

`1.0.0-rc.1` introduces typed LM Studio model discovery for OpenCode. It is
published under npm's `next` dist-tag while stable npm `latest` remains on
`0.3.1`.

Track rollout status, test coverage, and community feedback in
[#34](https://github.com/agustif/opencode-lmstudio/issues/34).

## Install

Pin the exact candidate in `opencode.json`:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": ["opencode-lmstudio@1.0.0-rc.1"]
}
```

The moving prerelease channel is available as `opencode-lmstudio@next`. To
return to stable, set the plugin entry to `opencode-lmstudio@0.3.1` and restart
OpenCode.

## Capabilities

- Validates LM Studio's `GET /api/v0/models` response at runtime.
- Adds `llm` models with text input/output metadata.
- Adds `vlm` models with text/image input and attachment support.
- Keeps embedding models outside OpenCode's chat-model list.
- Maps `max_context_length` into OpenCode context and output limits.
- Supports explicitly configured model overrides and whitelists.
- Supports provider `apiKey`, `{env:NAME}`, and private-network environment
  token sources.
- Allows LM Studio to load downloaded models on demand.
- Reports discovery through OpenCode's structured application logger.

## Compatibility

- OpenCode compatible with `@opencode-ai/plugin` 1.17.x
- LM Studio 0.3.6 or newer
- `GET /api/v0/models` available on the configured LM Studio server
- Node.js `22.22.2`, `24.15.0`, or a supported version from `26` onward; or
  Bun `1.3.5` or newer

Model IDs remain unchanged. Explicit names, limits, provider URLs, API keys,
model entries, and whitelists take precedence over discovered values.
